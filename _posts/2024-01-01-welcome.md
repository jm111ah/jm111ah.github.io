## 프로젝트 개요

GitHub Pages와 Jekyll을 활용하여 정적 사이트 생성 기술 블로그를 구축했습니다. 이 프로젝트는 마크다운 기반의 콘텐츠 작성과 자동화된 배포 프로세스를 통해 효율적인 블로그 운영을 목표로 합니다.

## 기술 스택

- **Jekyll**: 정적 사이트 생성기 (Ruby 기반)
- **GitHub Pages**: 무료 웹 호스팅 서비스
- **Git**: 버전 관리 시스템
- **Markdown**: 콘텐츠 작성 언어
- **Liquid**: 템플릿 엔진
- **HTML/CSS**: 프론트엔드 구조 및 스타일링

## 프로젝트 구조

```
Blog/
├── _config.yml          # Jekyll 설정 파일
├── Gemfile              # Ruby 의존성 관리
├── _layouts/            # HTML 레이아웃 템플릿
│   ├── default.html     # 기본 레이아웃
│   └── post.html        # 포스트 레이아웃
├── _posts/              # 블로그 포스트 디렉토리
├── assets/              # 정적 파일 (CSS, 이미지 등)
│   └── css/
│       └── main.css     # 커스텀 스타일시트
├── index.html           # 메인 페이지
├── blog.html            # 블로그 목록 페이지
└── about.md             # 소개 페이지
```

## 구현한 주요 기능

### 1. Jekyll 기반 정적 사이트 생성

Jekyll을 사용하여 마크다운 파일을 자동으로 HTML로 변환하는 정적 사이트를 구축했습니다.

**주요 구성 요소:**
- **Front Matter**: YAML 형식의 메타데이터 (제목, 날짜, 카테고리, 태그)
- **Layouts**: 재사용 가능한 HTML 템플릿
- **Posts**: 날짜 형식 파일명을 가진 마크다운 포스트
- **Pages**: 일반 웹 페이지

### 2. GitHub Pages 자동 배포

GitHub 저장소에 push하면 자동으로 GitHub Pages에 배포되는 워크플로우를 구축했습니다.

**배포 프로세스:**
1. 로컬에서 마크다운 파일 작성
2. Git으로 커밋 및 push
3. GitHub Pages가 자동으로 Jekyll 빌드 및 배포

### 3. 커스텀 레이아웃 시스템

기본 Jekyll 테마를 확장하여 커스텀 레이아웃을 구현했습니다:
- 포스트 전용 레이아웃 (`post.html`)
- 기본 페이지 레이아웃 (`default.html`)
- 네비게이션 및 메타데이터 표시

### 4. Git 기반 버전 관리

모든 콘텐츠와 설정을 Git으로 관리하여:
- 변경 이력 추적
- 협업 가능한 구조
- 롤백 및 브랜치 관리

## 기술적 도전과 해결

### 1. GitHub 인증 설정

**문제**: GitHub에 push하기 위한 인증이 필요했습니다.

**해결**: Personal Access Token을 사용한 HTTPS 인증 방식을 구현했습니다.
- `repo` 권한을 가진 토큰 생성
- 원격 저장소 URL에 토큰 포함하여 인증

### 2. Jekyll 설정 최적화

**구현 내용**:
- `_config.yml`에서 사이트 메타데이터 관리
- Kramdown 마크다운 파서 설정
- Rouge 코드 하이라이터 설정
- 플러그인 설정 (jekyll-feed, jekyll-sitemap)

### 3. 프로젝트 구조 설계

**고려 사항**:
- Jekyll 컨벤션 준수
- 확장 가능한 디렉토리 구조
- 정적 파일 관리 최적화

## 프로젝트의 특징

### ✅ 장점

1. **무료 호스팅**: GitHub Pages를 통한 무료 웹 호스팅
2. **빠른 로딩 속도**: 정적 파일만 제공하여 빠른 성능
3. **SEO 친화적**: 정적 HTML로 검색 엔진 최적화 용이
4. **버전 관리**: Git을 통한 콘텐츠 버전 관리
5. **마크다운 작성**: 간편한 마크다운 문법으로 콘텐츠 작성
6. **보안**: 데이터베이스 없이 정적 파일만 제공하여 보안 위험 최소화

### 📝 콘텐츠 작성 프로세스

1. `_posts` 폴더에 날짜 형식의 마크다운 파일 생성 (`YYYY-MM-DD-제목.md`)
2. Front Matter에 메타데이터 작성 (layout, title, date, categories, tags)
3. 마크다운 형식으로 콘텐츠 작성
4. Git으로 커밋 및 push
5. 자동 배포 완료
